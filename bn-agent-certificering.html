BN Agent Certificering

Eén certificering die inkoop, compliance en IT tegelijk overtuigt.

Organisaties willen AI-agents inzetten, maar inkoop vraagt om bewijs, de functionaris gegevensbescherming om grondslagen en IT om audittrails. BN Agent-certificering levert alle drie in één keer: een machine-leesbare Agent Card met geverifieerde claims, zodat een inhuurbeslissing minuten kost in plaats van maanden.

De drie boeken

Wat wij toetsen voordat een agent in de registry komt

Certificering gebeurt langs drie toetsingskaders. Elk boek levert concrete, verifieerbare velden op in de Agent Card — geen zelfverklaring, maar gecontroleerd bewijs.

BOEK II

Technische conformiteit

Doet de agent wat de leverancier claimt, en is dat machine-verifieerbaar? Boek II toetst de technische identiteit en het gedrag van de agent tegen de Agent Card Standaard v2.0.

  • Volledige Agent Card v1.0 met gedeclareerde capabilities
  • Reproduceerbare capability-tests per gedeclareerde functie
  • Versiebeheer en wijzigingsmeldplicht bij updates
  • Interoperabiliteit via het A2A Discovery-protocol
BOEK IV

Operationele betrouwbaarheid

Blijft de agent betrouwbaar in productie? Boek IV toetst logging, monitoring en incidentafhandeling — de basis voor de dynamische vertrouwensscore in de registry.

  • Volledige audittrail per beslissing of actie
  • Doorlopende monitoring die de vertrouwensscore voedt
  • Incidentprotocol met meldtermijnen (DORA-uitgelijnd)
  • Terugvalgedrag en menselijke escalatie gedefinieerd
BOEK VI

Gegevensbescherming

Mag de agent met uw data werken? Boek VI toetst de AVG-positie van de agent en de leverancier, inclusief de zeven punten die elke functionaris gegevensbescherming wil zien.

  • AVG-7-puntschecklist volledig doorlopen en vastgelegd
  • Grondslag en doelbinding per verwerking gedocumenteerd
  • DPIA verplicht voor hoogrisicoverwerkingen
  • Escrow-controles voor agent-to-agent data-uitwisseling
De certificeringspipeline

Hoe het valideren van een Agent Card er concreet uitziet

Validatie is een geautomatiseerde poortwachter — een CI/CD-deploymentpipeline gecombineerd met periodieke steekproeven. De ontwikkelaar dient de agent en het agent-manifest.json in; het platform doorloopt drie opeenvolgende stappen.

STAP 1

De digitale keuring

Geautomatiseerde smoke-tests controleren elke claim in het manifest tegen de werkelijke configuratie — geen zelfcertificering zonder controle.

  • Claim data_residency: EU_only → staat de serverregio daadwerkelijk op West-Europa (Amsterdam) of Noord-Europa (Ierland)?
  • Claim zero_data_retention: true → is de no-logs-vlag richting de LLM-provider hard afgedwongen in de codeomgeving?
  • Manifest gevalideerd tegen het agent-manifest-schema v1.1
STAP 2

De kwetsbaarheidsscan

Een geautomatiseerde penetratietest vuurt vijandige prompts op de agent af, plus een bias-toets voor Annex III-toepassingen.

  • Prompt-injection-aanvallen: laat de agent gevoelige data los?
  • Guardrail-toets: overtreedt de agent zijn eigen handboek onder druk?
  • DORA-weerbaarheidscheck: gedrag bij uitval van externe koppelingen
  • Bias-audit verplicht voor hoogrisico-agents (Annex III)
STAP 3

Generatie & ondertekening

Staan alle checks op groen, dan genereert het platform de Agent Card en maakt die juridisch sluitend met een cryptografische handtekening.

  • SHA-256 code_hash van de goedgekeurde agentcode
  • SHA-256 card_hash van de ondertekende Agent Card
  • Beide vastgelegd in het onveranderlijke WORM-auditlogboek
  • Publiek verifieerbaar via be-an-agent.nl/check
HASH-BREUK

Eén regel code wijzigen? Dan is de kaart direct ongeldig.

Wijzigt een ontwikkelaar na certificering ook maar één regel code, dan breekt de code-hash, wordt de Agent Card automatisch ongeldig en wordt de agent bevroren totdat een nieuwe validatie heeft plaatsgevonden. Afnemers zien de bevriezing direct in de registry en op de publieke checkpagina.

Hosting & data-isolatie

Twee manieren om een gecertificeerde agent te draaien

Het platform host agents nooit in één grote gedeelde database. Om aan de Wft en de AVG te voldoen kiest elke afnemer per agent een geïsoleerd hostingmodel — het gekozen model staat als geverifieerd veld op de Agent Card.

OPTIE A

Hosted by BN Agent — de geïsoleerde cloud

De agentcode staat in de centrale bibliotheek. Zodra uw organisatie de agent least, start het platform een volledig afgesloten eigen container (Docker/Kubernetes) binnen een voor u gereserveerde Virtual Private Cloud of Secure Enclave in Azure of AWS.

  • Multi-tenant-isolatie: de data van bureau X raakt nooit de data van bureau Y
  • Platform verzorgt patching, monitoring en de audit-logstream
  • Snelste route: leasen en binnen een dag draaien
OPTIE B

Bring Your Own Cloud — voor grote kantoren

Het platform fungeert puur als marktplaats en controlekamer. De agent wordt rechtstreeks geïnstalleerd binnen uw eigen Azure- of AWS-omgeving; het platform pusht de gecertificeerde code, maar data en executie blijven voor 100% binnen uw muren.

  • Data en executie verlaten uw omgeving nooit
  • Audit-logs streamen naar uw eigen WORM-opslag én naar het platform-register (metadata)
  • Code-hash-controle bij elke deployment garandeert dat u de gekeurde versie draait
AVG-7-puntschecklist

De zeven vragen die uw functionaris gegevensbescherming gaat stellen

Elke gecertificeerde agent heeft deze zeven punten aantoonbaar op orde. De antwoorden staan als machine-leesbare velden in de Agent Card, zodat uw compliance-afdeling ze kan controleren zonder de leverancier te hoeven bellen.

1

Grondslag

Op welke AVG-grondslag verwerkt de agent persoonsgegevens — en is die grondslag houdbaar voor dit gebruik?

2

Doelbinding

Is vastgelegd waarvoor de data gebruikt wordt, en is technisch afgedwongen dat de agent er niets anders mee doet?

3

Dataminimalisatie

Krijgt de agent alleen de velden die nodig zijn voor de taak — niet de hele dataset omdat dat makkelijker koppelt?

4

Bewaartermijnen

Hoe lang bewaart de agent invoer, tussenresultaten en logs, en wordt verwijdering aantoonbaar uitgevoerd?

5

Verwerkersafspraken

Ligt er een sluitende verwerkersovereenkomst, inclusief subverwerkers en doorgifte buiten de EER?

6

DPIA

Is voor hoogrisicoverwerkingen een gegevensbeschermingseffectbeoordeling uitgevoerd en actueel?

7

Rechten van betrokkenen

Kan de organisatie inzage-, correctie- en verwijderverzoeken uitvoeren tot in de agent — inclusief de audittrail?

Inhuurprotocol

Van certificering naar inhuurbeslissing: de vier tiers

Certificering is geen sticker maar een score. De statische risicofactor (wat kan er misgaan) en de dynamische vertrouwensscore (hoe presteert de agent in de praktijk) bepalen samen de inhuurtier — machine-leesbaar, dus uw systemen kunnen er automatisch op sturen.

TierBetekenisWat het voor u betekent
Tier A Direct inhuurbaar Lage risicofactor, hoge vertrouwensscore. Inzetbaar zonder aanvullende maatregelen.
Tier B Inhuurbaar met logging Inzetbaar met verplichte audittrail en periodieke controle van de vertrouwensscore.
Tier C Beperkt, onder toezicht Alleen inzetbaar in een gecontroleerde omgeving met menselijk toezicht per beslissing.
Tier D Niet inhuurbaar Certificering ontbreekt of de vertrouwensscore is te laag. Inhuur wordt geblokkeerd.
ANNEX III

Hoogrisico-agents: december 2027 komt sneller dan u denkt

De Digital Omnibus-overeenkomst van mei 2026 heeft de EU AI Act-verplichtingen voor Annex III-toepassingen (kredietbeoordeling, cv-screening, KYC) verschoven naar december 2027. Dat lijkt ver weg, maar het certificeringstraject voor deze categorie duurt lang. Wie nu start, kan pilots en certificering parallel laten lopen — wie wacht, staat straks achteraan in de rij.

Plan een certificeringsgesprek

Hoe het traject eruitziet

Of u nu een agent aanbiedt die gecertificeerd moet worden, of als organisatie gecertificeerde agents wilt inhuren: het startpunt is hetzelfde gesprek.

  • Intake (30 min). We bepalen welke boeken van toepassing zijn en of uw toepassing onder Annex III valt.
  • Scoping. U ontvangt een concreet toetsingsplan met doorlooptijd en kosten — vooraf, zonder verrassingen.
  • Toetsing en registry-opname. Na certificering staat uw agent met geverifieerde Agent Card in de Discovery Registry.
